# projectA

사장님과 제임스가 함께 진행하는 개발 작업 저장소입니다.

## 기본 작업 흐름

1. 사장님이 만들고 싶은 기능을 자연어로 말합니다.
2. 제임스가 필요한 파일을 만들거나 수정합니다.
3. 실행, 테스트, 오류 확인을 합니다.
4. 변경사항을 Git에 커밋합니다.
5. GitHub로 push해서 백업합니다.

## 사장님이 요청할 때 좋은 방식

예시:

```text
로그인 화면을 만들고 싶어.
초보자도 이해할 수 있게 설명하면서 진행해줘.
완료되면 실행 방법도 알려줘.
```

또는:

```text
이 오류가 났어. 원인과 해결 방법을 쉽게 설명하고 고쳐줘.
```

## 중요한 원칙

- 큰 변경 전에는 현재 상태를 확인합니다.
- 삭제, 외부 발송, 공개 배포 같은 위험한 작업은 먼저 확인합니다.
- 코드는 가능한 한 작게 나눠서 만들고 검증합니다.
- 사장님이 직접 해야 하는 일은 초보자 기준으로 단계별 안내합니다.
